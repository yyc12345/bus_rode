## 开始之前
#### SA 文本分割和文本分隔符
在接下来的内容中，你可能会见到一些数据格式，他们被称为分割文本，准确地来说，他们原本应当是一个列表，通过指定分隔符的形式让他们强制变成了一个String形式，以方便传输。  
在后文中，这种数据格式称为 **SA** \(StringArray\)，当你见到它的时候，您需要按以下方式将一个有序文本组织成单行并写入。由于只能是单行，故分隔符不能是换行符，换行符也不能出现在要编码的文本列表中任何一个文本里，否则会导致编码和解码时不一致。  
###### 普通格式：  
>String1#String2#String3#...#StringN  

其中的 # 表示的是分隔符，可以任意替换为其余字符，可以是单个字符，也可以是多个字符。在bus\_rode中，我们一般使用 **,**\(英文逗号\)作为分隔符。这也就表明了**,**不能作为要编码文本的一部分，如果不采用 **,** 作为分隔符，会在标注中标明采用何种分隔符。  
###### 嵌套格式
>String1@String2#String3@...@StringN  

其中 @ 和 # 分别是2种任意格式的不一样的分隔符，我们称这种格式为嵌套格式，嵌套格式可以看做是一个大的SA，而大的SA中的每一项又是一个SA如此嵌套形成的格式。在读取时，先对整体使用分割，在对分割的每一项再使用分割即可。  
###### 实现：
**Kernel.Tools.StringGroup\(\)**  
该类提供存储这种数据和输出为多种格式的能力，在bus\_rode中，该类被广泛使用。  
具体实现方式是利用 **System.Text.RegularExpressions** 下的 **Regex.Split\(\,\,\)** 函数进行字符串分割。  

#### 阅读事项
* 注意，本文档中**每个格式表格**中每跨过**列表**的一行，表示需要在文件中输入一个**回车**，然后再写入下一行，如果不需要那么做，会特别注释  
如have\_bus.txt中，输入完LINE 1之后需要输入回车，再输入下一行LINE 2的内容，以此类推  
* 此外，格式表格中会有含有 **-** 的表格块，这表明该行不是需要编写的内容，仅仅是为了理解方便而刻意做的逻辑分割，使得表格看起来更有条理。同理，带有 **...** 或 **N** 标识的助记符，其表示其中的内容可按一定规律进行扩展，扩展方式一般记录在第一项中。
* 与其他文档（如bus\_rode Resources Package File Format）不同的是，格式表格中的值类型并非表示该值需要按比特写入，它仅仅表示这个项的格式，是一种约束格式，如一个Integer类型的项写入，如果要写入5，并**不**是要你写入：0101 \(BIN\)，而是让你**输入一个文本**：5。也就是说，无论值类型给定的是什么，你必须且只能输入一系列文本（按要求和给定值类型约束）而不是二进制数值。
* 本文档中所介绍的所有文档均需要使用**UTF8无BOM**格式进行编码，否则无法读取。但并不是可以为所欲为读取，在下面一点中将会点明如何读取。

## 索引文件统一格式
#### 介绍
索引文件是一类文件的总称。他们拥有固定的格式因而不针对单个文件单个进行分析。在后文中，索引文件的**代号**是\*\*\*Seek  
\*\*\*Seek中 \*\*\* 指的是主资源文档的名称，必须要替换为其对应的主资源文档\(Bus, Subway, Stop等\)的名称。因为本文档实际上是其对应的主资源文档的索引。可以快速寻查需要的资源的索引。  
在当前，需要\*\*\*Seek文件的主文档有4个：Bus, Stop, GuideLine, Subway，也就是说必须存在BusSeek, StopSeek, GuideLineSeek, SubwaySeek这4个索引文件

#### 文件格式
这是整个文件的编写格式，在接下来会阐述每个查找模块如何编写  

|助记符|值类型|描述|
|:---:|:---:|:---:|
|SEARCH\_BLOCK 1|N/A|每一个查找区块格式应按照下文的方式进行编码|
|SEARCH\_BLOCK 2|N/A||
|...|N/A||
|SEARCH\_BLOCK N|N/A||
|-|-|-|
|END|String=Chr\(13\)|这里需要输入 **回车** 来表明这个文件结束|

#### 查找区块格式
每个查找区块指示了文件中每一个记录项的位置，记录项的位置由FileStream.Position指定，当在主资源文档中读取时，使用FileStream.Seek\(\)确定读取位置，并使用Length字段来准确读取。

|助记符|值类型|描述|
|:---:|:---:|:---:|
|HEAD|String|标识头，该头在不同的索引文件中使用不同的内容作为标识。在Bus索引中，使用线路名作为头；在Stop索引中，使用站台名作为头；在GuideLine中，使用线路名作为头；在Subway索引中，使用站台名作为头。头必须是唯一的，头必须具有代表性指示其在主资源文档的位置|
|POSITION\_AND\_LENGTH|SA|表示查找HEAD需要到达的Position和需要读取的长度Length，具体格式见下文|

#### 本部分的SA格式
###### POSITION\_AND\_LENGTH
**分隔符：,**
>Position,Length

|助记符|值类型|描述|
|:---:|:---:|:---:|
|Position|Long|写入时写入的是上一次写入之后FileStream.Position的值，读取时读取之后作为FileStream.Seek\(\)自文件头的偏移量直接使用。 |
|Length|Integer|写入时通过将写入字符串用System.Text.Encoding.UTF8.GetBytes\(\)得到Byte数组之后取其Length写入，读取时读取之后作为FileStream.Read\(\)的读取长度直接使用|

#### 线路区块格式
本格式负责表述一个线路内部的各个信息，它决定了每一个线路的核心数据。  

|助记符|值类型|描述|
|:---:|:---:|:---:|
|NAME|String|线路名称，在线路名称后面要添加“路”字或者其他标识性字符,不能仅填写数字，例如： 7路(正确)  7(错误) ; 地铁1号线(正确) 1(错误)，而且，线路名中不得包含空格|
|-|-|-|
|MSG 1|String|5个描述文字串之一，你可以在这里输入一些有关线路的信息，在同一个资源包中，建议这些文字尽量保持所描述的内容一致|
|MSG 2|String|同上|
|MSG 3|String|同上|
|MSG 4|String|同上|
|MSG 5|String|同上|
|TYPE|Integer|线路种类，写入0表示这是公交，写入1表示这是一个地铁线路|
|WAIT|Integer|该值对于车辆的可等待性进行描述。0=极难等(30min以上1班) 1=难等(16-30min 1班) 2=适中(6-15min 1班)  3=易等(5-2min 1班) 4=极易等(2min 1班或更多班次)|
|START\_HOUR|Integer|4个Integer值，取值范围：0<=START\_HOUR<12 12<END\_HOUR\<24 0<=START\_MINUTE 和 END\_MINUTE<60  意义：START\_HOUR,END\_HOUR表示小时，START\_MINUTE,END\_MINUTE表示分钟 START\_HOUR,START\_MINUTE表示车辆始发时间，END\_HOUR,END\_MINUTE表示末班车时间|
|START\_MINUTE|Integer||
|END\_HOUR|Integer||
|END\_MINUTE|Integer||
|-|-|-|
|UP\_STOP|SA|上行线路站点，这里开始输入站台，如果有该上下行上的单向站台也要写，每行一个站台名，站台最少**2**个，最多不要超过**200**个。**注意：此区间的站台由首个站台到最后站台的顺序应为车辆上行线路经过站台顺序**|
|DOWN\_STOP|SA|下行线路站点这里开始输入站台，每行一个站台名，站台最少**2**个，最多不要超过**200**个。**注意：此区间的站台由首个站台到最后站台的顺序应为车辆下行线路经过站台顺序**|

TIPS：
>若为夜班线路（即运营时间跨过 午夜0点 导致发车时间大于收车时间）此时按照标准直接对号入座填写，无须刻意颠倒顺序，bus\_rode内部将会自行判断  

>上行线路的定义：如果当地公交网站已经规定何为上行，或者当地人民普遍认同何为上行，则必须依照当地情况编写。如若没有明确定义，则认为向北行驶的为上行，整条线路大致东西走向的，则向东为上行，环形线路的默认内圈的线路为上行  

>若出现只有单方向车次时，只有下行线路，请将下行线路颠倒顺序写在上行线路区块中，即上行线路区块中的站台数目不得为0，必须填写，若只是单方向车次，则下行线路区块直接省略填写，例如：  
aaa,bbb,ccc,ddd **<-上行**  
**\<- 下行，留空**

**分隔符：,**
>UpStop1,UpStop2,...,UpStopN

###### DOWN\_STOP
**分隔符：,**
>DownStop1,DownStop2,...,DownStopN

**次结构分隔符：#**

## Readme
#### 介绍
该文件可以简述这个资源文件的属性，同时，这个文件还将指示程序，这台计算机是否安装了可用的资源文件，否则，应用程序将无法启动

#### 文件格式

|助记符|值类型|描述|
|:---:|:---:|:---:|
|ADDRESS 1|String|表明了你所开发的资源所应用的城市，按照以下标准填写：**国家-省（州，区）-城市（县，市）** 如果您有开发这个城市所对应的实时车辆模块的想法的话，您需要保证 **城市（县，市）** 一栏必须与模块开发者商定好（如果是您自己开发模块可以免去）否则模块将不会被加载。本部分的内容必须全部用英文编写。|
|ADDRESS 2|String|本部分也是地址，只不过是上面的地址用本国的语言再重新写一遍而已，该部分与联网搜索最短路径的功能有关。其中 **C** 不可省略，否则这将会影响联机查询换乘规划的功能|
|DEVELOPER|String|在这里书写开发这套资源的人的姓名，或者组织名称|
|DATE|String|书写最后更新这套资源的时间，没有固定格式，至少**精确到日**|

## 必须文件目录
本文档提供过时和现在正在使用的内容，所以，对于某版本的bus\_rode，某些文件是必须的，某些文件过旧或过新从而废除会不受支持。本部分主要介绍各个版本所需要的必须文件，本文中的过时所强调的都是以最新版本而言的，旧的版本请在这里寻找必要的文件，或者翻旧版的资源开发手册（如果有的话）。  
注意，一些文件可能在新旧版本之中都存在，但其编写方式和表达的意思或许大相径庭，所以，最稳妥的方式就是让用户始终使用最新的bus\_rode，制作最新的资源，或者拥有一本旧版的资源编写手册。  
下面是不同版本的必须文件目录
